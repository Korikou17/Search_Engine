# 搜索引擎项目

基于 C++17 + muduo 网络库的全文搜索引擎，支持中文/英文网页抓取、去重、索引构建、关键词推荐与在线检索。

## 三大核心模块

### 1. 离线索引构建（indexer）

从原始语料中提取关键词、构建词典和倒排索引：

- **中文关键词**：jieba 分词 → 过滤停用词 → 只保留纯中文字符 → 统计词频
- **英文关键词**：转小写 → 过滤非字母字符 → 过滤停用词 → 统计词频
- **中文索引**：对字典中每个词，按字符建立倒排索引，支持前缀/模糊查询
- **网页索引**：XML 解析 → SimHash 去重 → BM25 权重计算 → 倒排索引

### 2. 在线搜索服务（server）

基于 muduo TcpServer 的并发搜索服务：

- 4 个 I/O 线程处理并发连接
- TLV 协议编解码解决 TCP 粘包
- 支持两种请求类型：
  - `KeywordReq` 关键词推荐：返回混合评分排序的候选词
  - `PageReq` 网页搜索：返回 BM25 排序 + KWIC 摘要的 Top-5 结果

### 3. 搜索客户端（client）

交互式命令行客户端：

- 直接输入中文 → 网页搜索（默认）
- `/k <关键词>` → 关键词推荐
- `/s <关键词>` → 网页搜索（显式）
- `/help` → 查看帮助
- `/quit` → 退出

## 🚀 核心优化

> 以下为搜索引擎三个核心阶段（离线索引 → 在线检索 → 结果展示）的关键优化。

### 优化一：分块哈希表加速 SimHash 去重

**问题**：原始去重对每篇新文档遍历全部已有文档进行汉明距离比较，时间复杂度 $O(n^2)$，10 万篇文档需 ~50 亿次比较。

**解决方案**：利用鸽巢原理进行分块哈希索引。

> 64 位 SimHash，汉明距离阈值 ≤ 3
> → 切分为 4 个 16 位块
> → 若两文档相似，至少有一块完全相同
> → 建 4 张哈希表，只和同块匹配的候选者比较

```
SimHash: | 16 bits | 16 bits | 16 bits | 16 bits |
          块 0      块 1      块 2      块 3

新文档 → 查 4 张表 → 候选集（极少数文档）→ 完整汉明距离比较
                           ↑ 其余 99%+ 的文档直接跳过
```

| 文档数 | 原方案（O(n²)） | 优化后（O(n)） |
|--------|----------------|----------------|
| 1,000 | ~50 万 | ~1,000 |
| 10,000 | ~5,000 万 | ~10,000 |
| 100,000 | ~50 亿 | ~100,000 |

### 优化二：BM25 权重算法替代 TF-IDF

**问题**：传统 TF-IDF 对高频词线性放大（出现 100 次得分就是 10 次×10），容易堆砌关键词刷分；长文档被过度惩罚。

**解决方案**：采用工业界标准 **BM25** 算法，引入词频饱和与灵活的长度归一化。

$$score(D, Q) = \sum_{i=1}^{n} IDF(q_i) \cdot \frac{tf(q_i, D) \cdot (k_1 + 1)}{tf(q_i, D) + k_1 \cdot \left(1 - b + b \cdot \frac{|D|}{avgdl}\right)}$$

| 参数 | 默认值 | 作用 |
|------|--------|------|
| $k_1$ | 1.2 | 词频饱和控制（出现 10 次 vs 1 次 ≈ 5 倍，而非 10 倍） |
| $b$ | 0.75 | 长度归一化（0=不惩罚长文档，1=完全归一化） |

| 维度 | TF-IDF | BM25 |
|------|--------|------|
| 词频处理 | 线性增长 | 对数饱和 |
| 文档长度 | 简单除法 | $k_1$ + $b$ 双参数灵活调节 |
| 归一化 | RMS（需额外计算） | 内置，无需额外步骤 |
| 工业应用 | 教学/简单场景 | Elasticsearch / Lucene 默认算法 |

### 优化三：混合信号关键词推荐（在线推荐阶段）

**问题**：原方案采用纯粹的**编辑距离（Levenshtein）**衡量相关性，存在根本性缺陷：

| 查询 | 编辑距离相近但不相关的词 | 编辑距离远但高度相关的词 |
|------|--------------------------|--------------------------|
| 人工智能 | 人工呼吸（dist=2）✅ 推荐 | 深度学习（dist=4）❌ 不推荐 |
| 机器学习 | 机器翻译（dist=2）✅ 推荐 | 神经网络（dist=4）❌ 不推荐 |

编辑距离衡量的是**字符串拼写相似度**，而非**语义相关性**。

**解决方案**：采用工业界"相关搜索"的主流架构 — **多路召回 + 混合精排**。

#### 多路召回架构

```
用户输入 "人工智能"
       │
       ├─ 召回A：字符重叠召回（字符倒排索引）
       │   └─ 分解为 "人""工""智""能" → 找包含任一字的词
       │       → 保证召回率，不会漏掉相关词
       │
       ├─ 召回B：文档共现召回（BM25 倒排索引）
       │   └─ 查 inverted_index → "人工智能"出现在哪些文档 →
       │      这些文档中还有哪些词 → 共现词即语义相关词
       │       → 提供语义相关性信号
       │
       └─ 召回C：前缀匹配召回（Trie 前缀树）
           └─ 沿 Trie 逐字下钻 "人"→"工"→...
               → 精确召回以该前缀开头的词（输入提示补全）
```

#### 混合精排公式

$$score = 0.5 \times \text{Jaccard}_{docs}(Q, C) + 0.3 \times \left(1 - \frac{edit\_dist}{\max\_len}\right) + 0.2 \times \frac{\log_2(1+freq)}{\log_2(1+max\_freq)}$$

| 信号 | 权重 | 计算方式 | 设计意图 |
|------|------|----------|----------|
| 文档共现 Jaccard | 50% | $\frac{\|docs(Q) \cap docs(C)\|}{\|docs(Q) \cup docs(C)\|}$ | 两词出现在同一批文档中 → 语义相关（黄金标准） |
| 编辑相似度 | 30% | $1 - edit\_dist / \max\_len$ | 拼写变体/输入纠错 |
| 对数词频 | 20% | $\log_2(1+freq) / \log_2(1+max\_freq)$ | 热门词流行度辅助信号 |

### 优化四：渐进式放宽检索（在线检索阶段）

**问题**：原方案使用严格的 **AND 语义**——所有查询关键词都必须命中，任意一个不在索引中或交集为空就返回 0 结果。

```
用户输入 "GPT 大模型 应用"
  → "GPT" 不在索引中 → 直接返回 0 结果 ❌
```

**解决方案**：仿照 Google/Elasticsearch 的渐进式放宽策略，三轮降级兜底。

```
第 1 轮：严格 AND（所有关键词命中）
  ├─ 结果 ≥ 5 → 直接返回 ✅
  └─ 结果 < 5 → 进入第 2 轮

第 2 轮：≥60% 关键词命中
  └─ ceil(已知关键词数 × 60%) 个词命中即可
      ├─ 结果 ≥ 5 → 返回 ✅
      └─ 结果 < 5 → 进入第 3 轮

第 3 轮：OR（命中任意关键词）
  └─ 兜底策略，保证永远有结果返回
```

| 查询示例 | 旧算法 | 新算法 |
|----------|--------|--------|
| "GPT 大模型 应用" | 0 结果（GPT 不在索引中） | ≥60% = 2/3 命中 → 返回"大模型 应用"的结果 |
| "人工智能 机器学习 深度学习" | AND 返回充足结果 | 第1轮 AND 直接返回（行为不变） |
| "中国 发展"（交集可能为空） | 0 结果 | 第3轮 OR 兜底返回 |

### 优化五：关键词上下文摘要 KWIC（结果展示阶段）

**问题**：原方案直接取文档内容的前 50 个 UTF-8 字符作为摘要。用户搜索"深度学习"，摘要显示的是"本网站致力于提供最新的技术资讯..."——完全不包含查询词，毫无参考价值。

**解决方案**：采用 **KWIC（Keyword in Context）** 算法，以查询词命中位置为中心生成上下文摘要。

```
       命中位置
          ↓
  ────[前 25 字]───查询词───[后 25 字]────
  ...Transformer 是一种革命性的 深度学习 架构，它抛弃了传统的...
```

**多个关键词命中时自动合并重叠窗口**：

```
命中1 窗口: ┌──────────────────────┐
命中2 窗口:        ┌──────────────────────┐
                     ↑ 重叠 → 自动合并
合并结果:   ┌─────────────────────────────────┐
  "...人工智能技术飞速发展，其中深度学习作为人工智能的核心分支..."
```

算法步骤：
1. 构建 UTF-8 字符偏移索引，支持 byte offset ↔ char index 互转
2. 遍历所有查询关键词，找到正文中全部命中位置
3. 每个命中位置扩展为 ±25 字的窗口
4. 按起始位置排序 → 合并重叠窗口 → 选命中关键词最多的窗口
5. 截取文本 + 首尾添加 `...` 省略号

| 场景 | 旧摘要 | 新摘要 |
|------|--------|--------|
| 单关键词 | 文章开头前 50 字（无关） | 查询词前后各 25 字的上下文 |
| 多关键词（分散） | 文章开头前 50 字（无关） | 选取命中关键词最多的窗口 |
| 多关键词（相邻） | 文章开头前 50 字（无关） | 合并重叠窗口，一次性展示所有命中 |
| 关键词不在正文中 | 文章开头前 50 字 | 回退到前 50 字 + "..." |

## 编译与运行

```bash
# 编译
cd SearchEngine
bash build.sh

# 第一步：离线构建索引（必须先执行，生成 bin/data/ 下的索引文件）
cd bin
./offline_indexer

# 第二步：启动搜索服务
./server

# 第三步：新终端启动客户端
./client
# 直接输入中文 → 网页搜索
# /k <关键词>  → 关键词推荐
# /help        → 查看帮助
# /quit        → 退出
```
