# 搜索引擎项目

基于 C++17 + muduo 网络库的全文搜索引擎，支持中文/英文网页抓取、去重、索引构建、关键词推荐与在线检索。

## 三大核心模块

### 1. 离线索引构建（indexer）

从原始语料中提取关键词、构建词典和倒排索引：

- **中文关键词**：jieba 分词 → 过滤停用词 → 只保留纯中文字符 → 统计词频
- **英文关键词**：转小写 → 过滤非字母字符 → 过滤停用词 → 统计词频
- **中文索引**：对字典中每个词，按字符建立倒排索引，支持前缀/模糊查询
- **网页索引**：XML 解析 → SimHash 去重 → BM25 权重计算 → 倒排索引

### 2. 在线搜索服务（server）

基于 muduo TcpServer 的并发搜索服务：

- 4 个 I/O 线程处理并发连接
- TLV 协议编解码解决 TCP 粘包
- 支持两种请求类型：
  - `/w` 关键词推荐：返回编辑距离最近的候选词
  - `/p` 网页搜索：返回 BM25 排序后的 Top-5 结果

### 3. 搜索客户端（client）

交互式命令行客户端，支持三种操作：

- `/w` 触发关键词推荐
- `/p` 触发网页搜索
- `/quit` 退出

## 🚀 核心优化

### 优化一：分块哈希表加速 SimHash 去重

**问题**：原始去重对每篇新文档遍历全部已有文档进行汉明距离比较，时间复杂度 $O(n^2)$，10 万篇文档需 ~50 亿次比较。

**解决方案**：利用鸽巢原理进行分块哈希索引。

> 64 位 SimHash，汉明距离阈值 ≤ 3
> → 切分为 4 个 16 位块
> → 若两文档相似，至少有一块完全相同
> → 建 4 张哈希表，只和同块匹配的候选者比较

```
SimHash: | 16 bits | 16 bits | 16 bits | 16 bits |
          块 0      块 1      块 2      块 3

新文档 → 查 4 张表 → 候选集（极少数文档）→ 完整汉明距离比较
                           ↑ 其余 99%+ 的文档直接跳过
```

| 文档数 | 原方案（O(n²)） | 优化后（O(n)） |
|--------|----------------|----------------|
| 1,000 | ~50 万 | ~1,000 |
| 10,000 | ~5,000 万 | ~10,000 |
| 100,000 | ~50 亿 | ~100,000 |

### 优化二：BM25 权重算法替代 TF-IDF

**问题**：传统 TF-IDF 对高频词线性放大（出现 100 次得分就是 10 次×10），容易堆砌关键词刷分；长文档被过度惩罚。

**解决方案**：采用工业界标准 **BM25** 算法，引入词频饱和与灵活的长度归一化。

$$score(D, Q) = \sum_{i=1}^{n} IDF(q_i) \cdot \frac{tf(q_i, D) \cdot (k_1 + 1)}{tf(q_i, D) + k_1 \cdot \left(1 - b + b \cdot \frac{|D|}{avgdl}\right)}$$

| 参数 | 默认值 | 作用 |
|------|--------|------|
| $k_1$ | 1.2 | 词频饱和控制（出现 10 次 vs 1 次 ≈ 5 倍，而非 10 倍） |
| $b$ | 0.75 | 长度归一化（0=不惩罚长文档，1=完全归一化） |

| 维度 | TF-IDF | BM25 |
|------|--------|------|
| 词频处理 | 线性增长 | 对数饱和 |
| 文档长度 | 简单除法 | $k_1$ + $b$ 双参数灵活调节 |
| 归一化 | RMS（需额外计算） | 内置，无需额外步骤 |
| 工业应用 | 教学/简单场景 | Elasticsearch / Lucene 默认算法 |

## 编译与运行

```bash
# 编译
cd SearchEngine
bash build.sh

# 第一步：离线构建索引（必须先执行，生成 bin/data/ 下的索引文件）
cd bin
./offline_indexer

# 第二步：启动搜索服务
./server

# 第三步：新终端启动客户端
./client
# 输入 /w 触发关键词推荐
# 输入 /p 触发网页搜索
# 输入 /quit 退出
```
